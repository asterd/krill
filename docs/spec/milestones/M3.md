# M3 - A2A + Cooperative Agent Orchestration

## Status

- Priority: `P0`
- Depends on: M0, M1, M2
- Blocks: M4-M6

## Objective

Add A2A ingress and minimum viable cooperative multi-agent orchestration.

## In Scope

1. Protocol plugin `a2a` with strict envelope validation.
2. Cooperative orchestration mode:
   - `router`
   - `specialist`
   - `synthesizer`
3. Handoff policy:
   - max hops
   - allowed pairs
   - timeout per step
   - token/workflow budget
4. Cross-agent audit and OTEL trace continuity.
5. Declarative OrgSchema support (roles, handoff, escalation) as orchestration source.

## Out of Scope

- autonomous planner graphs with arbitrary depth
- UI workflow builder

## Required Implementation Steps

1. Add orchestration mode selector to agent/workflow config.
2. Add declarative OrgSchema model and validator.
3. Implement OrgSchema-to-runtime compilation for cooperative pipeline execution.
4. Add handoff command model and policy evaluator bound to schema constraints.
5. Persist handoff chain in workflow state.
6. Enforce hard stop on policy limits.
7. Ensure replies aggregate via synthesizer for client output.

## Required Tests

1. Unit tests:
   - policy evaluator (hop/budget/allowed-pair/timeout)
2. Integration tests:
   - two cooperative workflow scenarios end-to-end
   - one declarative OrgSchema-driven workflow scenario
3. Property tests:
   - no infinite handoff loops under bounded policy
4. Non-regression:
   - single-agent mode unchanged

## Exit Criteria

1. Cooperative mode stable for at least 2 real scenarios.
2. Policy violations are blocked and audited.
3. Trace spans preserve same workflow/trace context across agents.
4. Declarative schema execution matches expected orchestration behavior.

## LLM Delivery Checklist

1. Sequence diagrams (router/specialist/synthesizer flows)
2. Policy decision examples
3. Test and coverage summary
