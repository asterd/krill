# M6 - Enterprise Control Plane

## Status

- Priority: `P1`
- Depends on: M0-M5
- Blocks: none

## Objective

Provide centralized enterprise operations for protocols, agents, skills, policy, scheduling, sessions, and observability.

## In Scope

1. Control plane API:
   - protocol management
   - agent profile management
   - skill lifecycle management
   - policy/capability management
   - scheduler management
   - session diagnostics
2. Tenant and RBAC model.
3. Secrets rotation and audit retention policies.
4. Health, readiness, and diagnostics endpoints.
5. Operational runbook.
6. Helm chart packaging and cluster startup scripts.
7. Installation guides for Kubernetes and OpenShift.

## Out of Scope

- full-featured GUI (API-first scope)
- enterprise billing

## Required Implementation Steps

1. Define control plane authn/authz contracts.
2. Add CRUD APIs for manageable entities.
3. Add immutable audit events for control operations.
4. Add safe rollout semantics:
   - dry run
   - staged apply
   - rollback
5. Add operator diagnostics for incident response.
6. Deliver Helm chart under `deploy/charts/krill` with values overlays (`dev`, `minikube`, `prod`).
7. Add scripts:
   - `deploy/scripts/k8s/up-minikube.sh`
   - `deploy/scripts/k8s/install.sh`
   - `deploy/scripts/k8s/uninstall.sh`
8. Add installation docs:
   - `deploy/docs/k8s.md`
   - `deploy/docs/openshift.md`
9. Ensure cluster path deploys the same logical services as local compose path.

## Required Tests

1. Unit tests:
   - RBAC and authz logic
   - API validation and defaults
2. Integration tests:
   - end-to-end control-plane operations to data-plane effect
3. Security tests:
   - unauthorized operation rejection
   - audit tamper checks
4. Non-regression:
   - data plane keeps functioning with control plane disabled/failing
5. Deployment tests:
   - Helm lint/template validation
   - mini-kube install/uninstall smoke
   - OpenShift compatibility checks (manifest/security context)

## Exit Criteria

1. Core operational workflows manageable without process restart.
2. RBAC and audit verified by tests.
3. Runbook complete and validated in a simulated incident drill.
4. Helm + mini-kube flow works from one script.
5. Kubernetes and OpenShift install docs are complete and validated.

## LLM Delivery Checklist

1. API surface summary
2. RBAC matrix
3. Runbook references
4. Final quality gate evidence
