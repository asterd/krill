# M0 - Compatibility Foundation

## Status

- Priority: `P0`
- Depends on: none
- Blocks: M1-M6

## Objective

Introduce versioned contracts (`EnvelopeV2` + config compatibility) without behavior regressions.

## In Scope

1. `EnvelopeV2` data model and defaults.
2. `v1 <-> v2` mapper.
3. Plugin compatibility layer for `http`, `telegram`, `webhook`.
4. Config compatibility matrix and validation.
5. Local development bootstrap foundation (`docker`, `docker-compose`, `docker-sandbox`).

## Out of Scope

- new protocols
- cooperative orchestration
- persistent storage changes

## Required Implementation Steps

1. Add envelope schema/version package (`internal/schema` or equivalent).
2. Implement strict mapper with roundtrip stability.
3. Integrate normalizer in ingress pipeline before bus publish.
4. Preserve existing external request/response payloads.
5. Add feature flag for strict v2 validation mode.
6. Add local startup scripts:
   - `deploy/scripts/dev/up.sh`
   - `deploy/scripts/dev/down.sh`
   - `deploy/scripts/dev/reset.sh`
7. Add compose files:
   - `deploy/compose/docker-compose.yml`
   - `deploy/compose/docker-compose.sandbox.yml`
8. Ensure one-command startup boots Krill + required local dependencies + sandbox profile.

## Required Tests

1. Unit tests:
   - field defaulting
   - mapper conversion
   - invalid schema rejection
2. Golden tests:
   - sample v1 payloads remain accepted
   - v2 payloads normalize deterministically
3. Non-regression tests:
   - current `http`, `telegram`, `webhook` flows unchanged
4. Performance:
   - micro-benchmark mapper overhead
5. Dev bootstrap smoke tests:
   - `up.sh` reaches healthy state
   - `down.sh` fully stops stack
   - `reset.sh` recreates clean environment

## Exit Criteria

1. No protocol behavior regression.
2. Roundtrip mapping deterministic.
3. p95 latency impact <= +5%.
4. Coverage contract satisfied.
5. Local stack starts with one command and supports end-to-end chat smoke test.

## LLM Delivery Checklist

1. Code changes
2. Test artifacts
3. Coverage report
4. Regression summary
5. Risks/follow-up notes
