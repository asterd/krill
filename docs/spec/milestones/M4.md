# M4 - Cron Scheduler + Long-Running Sessions

## Status

- Priority: `P0`
- Depends on: M0, M1, M2, M3
- Blocks: M5-M6

## Objective

Enable scheduled agent execution and persistent long-running development sessions for backend usage.

## In Scope

1. Cron scheduler for agents/workflows.
2. Persistent session lifecycle (`open`, `resume`, `checkpoint`, `close`).
3. History retention and summarization policies.
4. Session recovery after restart.
5. Trace/audit coverage for scheduler and session operations.
6. Versioned context primitives (`branch`, `merge`, `commit`) for long sessions.

## Out of Scope

- distributed scheduler sharding (can be follow-up)
- advanced workflow calendar UI

## Required Implementation Steps

1. Add schedule model:
   - `schedule_id`, `cron_expr`, `timezone`, `target`, `payload_template`, `concurrency_policy`, `enabled`
2. Implement scheduler engine:
   - missed-run behavior
   - retry policy
   - concurrency policy (`allow|forbid|replace`)
3. Implement session API/service:
   - open/resume/checkpoint/close
4. Add history persistence with retention by tenant/project.
5. Add summarization checkpoint mechanism to bound prompt growth.
6. Add versioned context APIs:
   - `session.branch`
   - `session.merge`
   - `session.commit`
7. Implement conflict handling policy for context merge (`fail`, `last-write-wins`, `manual`).
8. Persist provenance metadata for branch/merge/commit events.

## Required Tests

1. Unit tests:
   - cron parsing and next-run computation
   - concurrency policy behavior
   - session state transitions
2. Integration tests:
   - scheduled workflow trigger -> execution -> audit
   - long session resume across process restart
   - branch/merge/commit session flow with deterministic replay
3. Non-regression:
   - unscheduled interactive flows unchanged
4. Time-sensitive tests:
   - deterministic clock injection (no flaky wall-clock tests)

## Exit Criteria

1. Scheduled runs execute deterministically under policy.
2. Session resume restores sufficient context for continuous coding workflows.
3. History retention and summarization policies are enforced.
4. Versioned context branch/merge behavior is auditable and replayable.

## LLM Delivery Checklist

1. Scheduler state machine
2. Session lifecycle diagram
3. Recovery and retention test evidence
